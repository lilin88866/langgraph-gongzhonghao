# 标题生成

## 使用场景

当用户需要为微信公众号文章、视频号联动文章、技术解释、教程、观点文或系列内容生成标题时使用。

## 输入信息

- `topic`：文章主题。
- `core_point`：最重要的观点、结论或读者收益。
- `target_audience`：目标读者。
- `tone`：语气风格，例如实用、专业、犀利、温和、教程型、小林风格等。
- `constraints`：限制要求，例如标题长度、禁用词、是否允许强钩子。

默认限制：

- 标题要自然可信。
- 避免“必看”“颠覆”“震惊”“100%”等夸张表达。
- 优先体现 AI 知识点、具体场景、学习收益和明确读者对象。
- 如果用户输入是一整句问题、需求说明或长提示词，必须先压缩成清晰短标题，不能把原问题原样作为标题。
- 标题不要出现“读者介绍”“适合所有朋友阅读”“并且对每个架构做 demo”这类需求描述句。
- 技术解释类标题建议控制在 12-28 个中文字符；必要时可用冒号补充副标题。
- 订阅号知识型标题优先使用“概念 + 场景/收益”的结构，例如“Agent工作流入门”“MCP到底怎么用”。

## 标题类型

按以下类型生成标题候选：

- `干货型`：突出实用价值。
- `痛点型`：从读者当前问题切入。
- `问题型`：提出读者关心的问题。
- `反常识型`：挑战常见误解。
- `对比型`：比较两种方法、工具或认知。
- `系列型`：适合长期栏目或连续选题。

## 输出格式

必须使用下面这个结构：

```markdown

干货型：
1. ...
2. ...

痛点型：
1. ...
2. ...

问题型：
1. ...
2. ...

反常识型：
1. ...
2. ...

对比型：
1. ...
2. ...

系列型：
1. ...
2. ...

### 推荐标题

[最推荐的标题]

### 推荐理由

[说明为什么这个标题最匹配文章内容、目标读者和平台场景。]

### 不建议使用的标题

1. [Bad title]：原因。
2. [Bad title]：原因。
```

## 评估规则

生成时按以下标准评估标题：

- 主题清晰。
- 读者收益具体。
- 不过度夸张。
- 有足够好奇心。
- 和正文内容一致。
- 能看出这是一篇 AI 知识解释或实用教程，而不是纯热点转述。
- 适合手机端预览长度。
- 不等于用户原始输入。
- 不像一句需求说明，而像一篇可发布文章的标题。

优先选择具体、可信、有信息量的标题，不要只追求戏剧化。
