# 图片配文与配图提示词

## 使用场景

当文章结构已经清楚，用户需要封面图方案、正文配图提示词、截图位置建议或图片配文时使用。正文配图建议应优先直接插入文章 HTML 对应段落后，文末再汇总 AIGC 提示词和版权提醒。

## 输入信息

- `topic`：文章选题。
- `recommended_title`：当前标题。
- `article_html`：文章正文。
- `summary`：文章摘要。
- `image_urls`：已有图片 URL，如有。
- `account_positioning`：账号定位、语气和视觉风格。

如果没有提供视觉风格，默认使用干净、克制、现代、实用、适合手机端的风格。

## 视觉规则

- 图片要匹配具体文章段落，不要只匹配标题。
- 优先选择清晰概念，不要堆砌复杂视觉隐喻。
- AI、工具、教程类文章可使用干净界面、流程图、桌面场景、示意图或抽象效率场景。
- 不要要求真实品牌 logo、名人肖像、受版权保护角色或误导性截图。
- 如果图片用于表达数据，在未经验证前要说明是“示意图”。
- 图片配文要短，并且对手机端阅读有帮助。
- 正文配图建议不要只放在文末；需要在正文相关段落后插入一个浅色提示卡片，说明图片类型、画面结构、用途和版权提醒，方便编辑直接按位置出图。

## 来源图片重画规则

当来源图片是流程图、架构图、表格截图、三栏对比图、路线图、系统框图时，必须按“重画逻辑”处理，不要做简单换风格、换滤镜或照搬布局。

执行流程：

1. **先提取信息结构**
   - 识别图片类型：流程图 / 架构图 / 表格截图 / 对比图 / 路线图 / 系统框图。
   - 提取核心对象、层级关系、箭头方向、对比维度、输入输出、关键结论。
   - 如果 `image_urls` 只提供了图片链接和 `alt/title`，只能基于链接说明、正文上下文和 alt 推断结构，不能声称已经看见图片细节。
   - 如果看不清文字、无法访问图片或图片缺失，要明确要求人工补充原图、截图 OCR 或文字说明。

2. **再改写成新的图文结构**
   - 保留原图表达的“信息关系”，不要保留原图的版式、配色、图标、排版和大段文字。
   - 把原图的结构重新组织成更适合公众号阅读的新图。
   - 例如：
     - 原图：`ReAct / Plan-and-Execute / Reflection` 三栏对比。
     - 新图：`三种 Agent 范式选型路径图`。
     - 原图：系统架构图。
     - 新图：从“用户请求 -> 核心模块 -> 输出结果”的简化流程图。
     - 原图：表格截图。
     - 新图：卡片式对比图或决策树。

3. **最后生成新图提示词**
   - 提示词要描述新图的主题、信息结构、布局、中文标题、视觉风格和尺寸。
   - 必须明确“重新绘制 / 信息重组 / 示意图”，避免生成近似原图。
   - 重点是“重画逻辑”，不是“换个滤镜”。

来源图片处理禁忌：

- 不要输出“保留原图布局，只换颜色”。
- 不要复刻原图图标、品牌标识、截图界面和文字排版。
- 不要把来源截图当作可直接发布图片。
- 不要声称 AIGC 新图和原图完全一致。

## 千问生成图片脚本

如果用户希望直接生成图片，可以使用当前目录下的千问 / 通义万相图片生成脚本：

```bash
python skills/image-caption-prompt/imagegen.py "公众号封面图，AI Agent 工作流，干净科技风" --size 1024x1024
```

脚本规则：

- 只调用千问 / DashScope 图片生成接口。
- 默认读取项目 `.env` 中的 `QWEN_API_KEY`，也支持 `DASHSCOPE_API_KEY` 或 `--api-key`。
- 默认模型为 `wan2.2-t2i-flash`；如需更高质量，可在 `.env` 中设置 `DASHSCOPE_IMAGE_MODEL=wan2.2-t2i-plus`。
- 默认输出目录为 `~/Downloads/QwenImages`。
- 推荐公众号封面使用 `1024x1024`。
- 推荐竖版正文配图使用 `1024x1536` 或 `1024x1792`。
- 如果只是生成提示词，不需要输出命令；如果用户说“生成图片”“出图”“做封面图”，必须给出可执行命令。

必须使用下面这个结构：

```markdown
### 封面图方案

1. 主题：
   画面：
   风格：
   中文字建议：
   AIGC 提示词：
   千问生成命令：

### 正文配图位置

1. 位置：
   配图目的：
   图片类型：示意图 / 截图 / 流程图 / 人物场景 / 对比图
   正文插入卡片：
   来源图片结构提取：
   新图文结构：
   AIGC 提示词：
   千问生成命令：
   图片配文：

### 已有图片使用建议

1. 图片：
   建议位置：
   配文：
   版权/来源提醒：
   是否建议重画：
   重画方向：
   需要人工补充：

### 不建议使用的图片

1. [图片方向或已有图片]：原因。

### 发布前图片检查清单

- [ ] 封面图和标题一致。
- [ ] 正文配图能帮助理解，不只是装饰。
- [ ] 截图、照片、图标、logo 有使用权或已替换。
- [ ] AIGC 图片没有制造虚假事实。
- [ ] 图片中文字清晰，适合手机端阅读。
```

## 质量检查

输出前检查：

- 封面图提示词足够具体，可用于生成图片。
- 正文配图建议对应真实文章段落。
- 图片配文简短，可以直接发布。
- 避免了版权敏感视觉元素。
