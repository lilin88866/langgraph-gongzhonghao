## 简介

公众号文案改写是一款公众号原创改写工具，内置文章结构、标题、来源复核和发布风险控制规则，帮助用户将素材转化为可人工发布的公众号草稿。Agent 以公众号编辑的角度进行改写，不说"帮你改写"。

通过简单的输入，你可以：
- 将素材转换为可信、清晰、有吸引力的公众号风格
- 自动生成不超过20字的推荐标题和标题候选
- 融入公众号高频关键词和金句
- 末尾附加5-10个以 # 开头的 Tags 标签

适用于公众号号主、内容运营、品牌编辑、自媒体创作者等需要高效产出可信公众号草稿的用户。

技术基础：基于公众号阅读体验、结构化写作、事实复核和风险控制规则，结合 AI 模型实现原创改写。

## 功能特性

### 核心功能
- **🎯 标题生成**：生成不超过20字、自然可信且有信息量的标题
- **✍️ 风格转换**：将文案转换为态度鲜明、犀利直接、金句频出的公众号风格
- **📝 结构优化**：分点说明、步骤教程、对比展示，结尾引导互动
- **🏷️ Tags 附加**：结尾自动添加5-10个以 # 开头的相关标签

### 特色亮点
- **⚡ 写作与复核原则**：覆盖标题、开头、观点、事实、来源、版权和发布风险等维度
- **💡 2-3个关键词**：从10个公众号高频关键词中智能选择并自然融入正文
- **✅ 输出自检清单**：11项检查清单确保每次输出质量
- **🧠 千问改写任务**：生成供千问执行的公众号改写任务说明，不调用外部统计接口

## 使用指南

### 核心原则
**Agent 以文案发起者的角度进行改写**——用户的文案就是 Agent 的文案，Agent 只做公众号文案风格转换。

### 基础使用（3步）

#### 第1步：提供文案
向 Agent 提供需要改写为公众号风格的文案内容：

```
帮我把这段改为公众号文章：最近天气变化大，很多人感冒了，大家注意保暖
```

#### 第2步：生成千问改写任务
脚本读取公众号规则，并生成供千问执行的改写任务说明：

```bash
python scripts/rewrite.py "最近天气变化大，很多人感冒了，大家注意保暖"
```

#### 第3步：获取改写结果
Agent 参考 `assets/platform-rules.md` 中的规则，并严格遵循以下创作规范输出结果。

### 行文原则（10项必守）

1. **根据优质公众号文章总结行为特征和内容**：分析文章结构、开头逻辑、读者收益和互动话术
2. **提炼优质内容规律**：从可信文章中提炼标题、内容、节奏和表达规律
3. **标题要简单粗暴**：直击痛点，引发好奇，让读者忍不住点开
4. **给大家一种不看就有损失的感觉**：制造信息差和紧迫感，让读者觉得错过就是损失
5. **观点能颠覆常识**：能提供看待世界的新鲜角度，或对当下的生活态度有所启发
6. **能表达情感共鸣**：能体察人性的痛点，表达大众的情感共鸣
7. **能结合经历充实文章**：融入真实案例和个人故事，增加可信度和代入感
8. **文章跟"我"有关**：核心问题落实到读者的关联上，在读者的角度思考问题
9. **文章对"我"有用**：提供实用价值，让读者觉得读完有收获
10. **文风足够有趣**：用幽默、犀利的方式表达，避免枯燥说教

### 风格特点

- **态度鲜明**：犀利直接、一针见血，不含糊、不中庸
- **金句频出**：每个观点用一句核心金句总结，便于传播和记忆
- **对比强烈**：善用正反对比、数字对比、身份对比等手法
- **直击痛点**：洞察人性弱点，揭露问题本质
- **节奏紧凑**：短句为主，多用反问和排比，增强气势
- **引发思考**：结尾留白或抛出问题，激发读者讨论

### 标题创作规范

- 必须参考优质公众号文章中自然可信的标题结构
- 优先使用**数字型标题**（如："3个技巧"、"5天见效"）
- 可使用疑问句引发好奇（如："为什么..."、"居然..."）
- 可使用感叹句增强情绪（如："太绝了！"、"绝了！"）
- **标题最多不超过20字**（包含标点符号，超出必须精简）

### 正文创作规范

- 优先贴近原文的版式骨架、信息顺序、小标题层级、列表/引用/步骤形态；不要把原文强行改成固定模板。
- 改写目标相似度控制在 **25%-30%**：低于 25% 说明改得太大，要恢复原文结构、信息顺序和关键表达；高于 30% 说明过于接近，要替换连续句式和局部表达。
- 不要大改选题角度、事实边界、论证路径和段落顺序。降重主要靠换句式、换类比、调整表达和补少量解释，不连续复用原文句子。
- 当原文结构松散、缺少解释或用户明确需要教程化时，再参考 `skills/xiaolin-rewrite/SKILL.md` 的“小林笔记式讲解”结构补充 `简要回答`、`详细解析`、`选型 / 使用建议`、`重点回顾` 等模块。
- 开头必须直接回答核心问题，不要写“本文将会”“接下来我会”这类空泛开场。
- 抽象概念必须用生活化类比解释，例如外卖骑手、项目经理、错题本、公司管理制度、员工干活方法等。
- 多概念对比时，必须讲清“各自解决什么问题”，不要简单写成谁更高级。
- 每个小节按“定义 -> 类比 -> 区别 -> 优势 -> 代价 -> 适用场景”展开，具体内容可按选题裁剪。
- 必须补充工程视角：成本、延迟、复杂度、维护难度、适用边界或风险提醒。
- 如果用户要求 demo，demo 必须演示当前主题的运行逻辑，不能只写“调用模型解释这个问题”。
- 禁止写成面试对话体；不要出现“面试官：”“我：”“候选人：”“上一页”“下一页”“关注公众号”等内容。
- **排版**：良好分段，少量 Emoji 即可，不要每段都加；使用简短句子，适合手机阅读。
- **Tags**：结尾添加5-10个相关标签，以 # 开头。

### 输出检查清单

改写完成后，逐项确认：

- [ ] 标题不超过20字，有吸引力/钩子
- [ ] 携带2-3个 Keyword，自然融入正文
- [ ] 口语化表达（亲切自然，避免书面语）
- [ ] 态度鲜明，犀利直接
- [ ] 有金句，便于传播
- [ ] 段落结构清晰，分段良好
- [ ] 结尾有引导互动话术
- [ ] 末尾有 Tags（5-10个，以 # 开头）
- [ ] 点缀少量 Emoji（每段1-2个）
- [ ] 以作者身份输出（不说"帮你改写"）
- [ ] 不询问用户，直接输出结果

## 使用场景

- **角色**：公众号号主
- **需求**：将某个话题或观点写成有新意、有依据的公众号文章
- **使用方式**：输入核心观点和素材
- **预期收益**：获得态度鲜明、金句频出、引发讨论的公众号爆文

### 场景2：产品软文转化
- **角色**：品牌内容运营
- **需求**：将产品介绍转化为公众号读者爱看的内容
- **使用方式**：输入产品信息和卖点
- **预期收益**：获得融入真实案例、有情感共鸣的推广软文

### 场景3：教程 / 指南类文章
- **角色**：知识付费创作者
- **需求**：将专业知识写成公众号风格的分步骤教程
- **使用方式**：输入知识要点和教程大纲
- **预期收益**：获得带"指南"/"教程"关键词、结构清晰的实用文章

### 场景4：热点话题借势营销
- **角色**：自媒体运营
- **需求**：快速借势热点话题输出公众号文章
- **使用方式**：输入热点话题和个人观点
- **预期收益**：获得紧跟热点、有观点、有互动的公众号内容

## 常见问答

### 安装相关
**Q: 需要安装什么依赖？**
A: 不需要额外依赖，使用 Python 标准库即可。

**Q: Python 版本要求？**
A: Python 3.x，脚本默认使用 `/usr/bin/python3`。

### 使用相关
**Q: 标题必须不超过20字吗？**
A: 是的，必须严格遵守，超出会自动精简。

**Q: 关键词从哪里选择？**
A: 从10个公众号高频关键词中（指南、教程、解析、秘籍、经验分享、挑战、故事、案例分析、趋势预测、重要事件）选择2-3个自然融入正文。

**Q: 改写会自动添加 emoji 吗？**
A: 会点缀少量 Emoji（每段1-2个），但不会过量使用。

### 故障排除
**Q: 还会上报记录吗？**
A: 不会。当前版本不再调用外部统计接口。

**Q: 实际由哪个模型改写？**
A: 实际改写由千问根据公众号规则完成。

### 安全与许可
**Q: 是否需要单独 API Key？**
A: 使用项目已有千问配置即可。
