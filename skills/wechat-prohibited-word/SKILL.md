# 公众号发布风险检查

## 简介

这个 Skill 用于公众号发布前自查。它基于千问分析文案风险，覆盖夸大承诺、绝对化用语、未经复核的数据、医疗金融法律高风险表述、版权和肖像风险等场景。

## 配置

使用项目已有千问配置：

```bash
QWEN_API_KEY=你的 DashScope / 千问 API Key
```

## 使用方式

### 直接检查文本

```bash
python scripts/check_sensitive_words.py --content "这款产品三天见效，绝对第一"
```

### 只提取文本

```bash
python scripts/check_sensitive_words.py --file /path/to/article.txt --extract-only
```

脚本会输出 JSON，供千问风险分析使用。

## 检查维度

千问需要检查：

1. 夸大承诺和绝对化表达。
2. 医疗、金融、教育、法律等敏感领域表述。
3. 未验证数据、排名、案例和产品功效。
4. 版权、肖像、品牌、截图来源风险。
5. 标题党、误导性表述和过度营销。

## 输出格式

```markdown
### 风险等级

低 / 中 / 高

### 必须修改

1. [原句]：问题说明；建议改为：[替换表达]

### 建议修改

1. [原句]：优化理由；建议改为：[替换表达]

### 可以保留

1. [内容]：保留理由

### 发布前复核

- [ ] 数据和来源已复核。
- [ ] 图片版权已复核。
- [ ] 没有绝对化、承诺收益或医疗功效暗示。
```

## 注意事项

- 千问风险检查不能替代法务审核。
- 不确定的事实必须标注“需要人工复核”。
- 不要编造法规条款、平台规则或官方结论。
