# 核心工作流程

本文档定义公众号发布前风险检查流程。当前版本使用千问进行文本风险分析，不调用外部敏感词接口。

## 输入

- `content`：待检查的标题、摘要、正文或发布文案。
- `platform`：默认公众号。
- `manual_notes`：用户补充的行业、产品、限制要求。

## 步骤1：文本提取

如果输入是文件或网页摘要，先提取纯文本。不要把 HTML 样式、脚本或导航内容当成正文。

## 步骤2：千问风险分析

请千问从以下维度检查：

1. 夸大承诺。
2. 绝对化用语。
3. 未验证数据。
4. 医疗、金融、法律等高风险表述。
5. 版权、肖像、品牌和截图使用风险。
6. 可能引发误导的标题或封面文字。

## 步骤3：输出修改建议

输出格式：

```markdown
### 风险等级

低 / 中 / 高

### 必须修改

1. [问题]：修改建议。

### 建议修改

1. [问题]：优化建议。

### 可以保留

1. [内容]：保留理由。

### 发布前检查清单

- [ ] 标题不过度夸张。
- [ ] 事实、数据、产品名和链接已复核。
- [ ] 图片版权和来源已复核。
- [ ] 没有绝对化、承诺收益或敏感表述。
```

## 注意事项

- 千问检查不能替代法务、平台规则或人工审核。
- 不确定的事实必须标记为需要人工复核。
- 不要声称已经调用外部敏感词数据库。
